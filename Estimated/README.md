# 練等進度紀錄 ES2022

這是一個不用資料庫、不用框架的 Vanilla ES2022 單頁工具。

## 功能

- 同一天可以輸入多筆紀錄，紀錄區會自動彙整成每日一筆；點「詳細」才展開當日明細與完整 EXP
- 練等分鐘可輸入 `0`；代表活動、任務、道具等其他渠道獲得，只記錄角色成長，不納入平均練等效率
- 目標設定新增三選一燃燒規則：無燃燒、燃燒 1+1（270）、燃燒 1+1（280）
- 經驗輸入框改用文字輸入 + 自動清理，修正手機輸入小數點時游標或小數點跑到前面的問題
- 只有「目標設定」會自動儲存；每日紀錄、清除紀錄、重置全部、刪除單筆都維持手動操作
- 統計區新增「每日最低時數」，用玩家目前平均 EXP / 小時計算：要在目標日前達成，每天最少需要練多久
- 統計區新增「每日平均練等」：尚無紀錄時使用預設值；開始輸入練等紀錄後，會用實際每日練等分鐘自動換算平均小時
- 統計區「剩餘需練時數」只顯示純小時數，不再轉成天數，避免誤會成現實日期
- 推算完成日會獨立顯示在「預估完成日」卡片；若預估完成日超過設定的目標結束時間，日期與說明會以警告色提示
- 新增「總 EXP 預估完成」卡片：使用每日總獲得 EXP 推算完成時間，包含練等與其他渠道成長

- 起始設定只輸入：起始等級、起始經驗、目標等級與燃燒規則；目標固定計算到該等級 0.000%
- 每日紀錄輸入結束等級、結束經驗與練等分鐘；結束經驗支援到小數點第三位
- 練等分鐘輸入 0 時，該筆 EXP 仍會推進總進度，但不計入累積練等時間、平均效率、剩餘需練時數
- 自動統計：
  - 目前進度
  - 距離目標 EXP
  - 完成比例
  - 累積練等時間
  - 平均 EXP / 小時
  - 剩餘純練等時數，小時制顯示
  - 每日平均練等小時
  - 獨立的「預估完成日」卡片，超過目標時間會警告
  - 依每日總獲得 EXP 推算的「總 EXP 預估完成」日期
  - 目標日前平均每日需要 EXP
  - 依目前平均效率推算的每日最低練等時數
- 黑暗模式 UI
- PC / 手機響應式版型
- 使用瀏覽器 localStorage 自動儲存；另外提供手動匯出 / 匯入 JSON 備份
- 清除紀錄、重置全部、刪除當日紀錄、刪除單筆明細

## 執行方式


因為此專案使用 ES modules，建議用本地伺服器開啟，不要直接雙擊 `index.html`。

### 方式一：Node.js

```bash
npm start
```

開啟：

```text
http://localhost:5173
```

### 方式二：Python

```bash
python -m http.server 5173
```

開啟：

```text
http://localhost:5173
```


## 儲存方式

這版已移除「本機 JSON 自動同步」功能，避免瀏覽器重新整理後反覆要求檔案授權。

目前儲存邏輯固定為：

```text
主要儲存：瀏覽器 localStorage，自動保存
備份方式：手動匯出 JSON
還原方式：手動匯入 JSON
```

玩家每次修改目標設定、新增紀錄、刪除紀錄、清除紀錄或重置資料，都會自動寫入目前瀏覽器。

注意事項：

```text
清除瀏覽器網站資料會刪除紀錄
換瀏覽器或換裝置不會自動同步
建議定期按「匯出備份」下載 JSON 保險
需要還原時按「匯入備份」選擇 JSON 檔即可
```

匯出的備份檔案是 JSON，不是 JS：

```text
程式本體：HTML / CSS / JS
玩家資料：level-tracker-backup-YYYY-MM-DD.json
```

備份內容會包含版本、儲存時間與實際資料，例如：

```json
{
  "version": 1,
  "savedAt": "2026-06-28T12:00:00.000Z",
  "app": "level-tracker-es2022",
  "storage": "browser-localStorage",
  "data": {
    "goal": {},
    "records": []
  }
}
```

## 計算單位

玩家畫面只需要輸入 0～100 的數字，不需要輸入 `%` 符號，例如：

```text
265 等，經驗輸入 34.567
```

程式內部會先轉成 1/1000% 單位：

```text
34.567 = 34.567% = 34567 / 100000
```

再用 BigInt 搭配你提供的每等升級 EXP 表計算實際 EXP，避免高等級大數字在加總時失真。

燃燒規則由目標設定三選一決定：

```text
無燃燒：260 → 261 → 262 → ... → 300，每等都計算
燃燒 1+1（270）：260 → 262 → 264 → 266 → 268 → 270，270 後每等計算
燃燒 1+1（280）：260 → 262 → 264 → ... → 278 → 280，280 後每等計算
```

燃燒 1+1 模式下，被跳過的奇數等級不會被計入需求，也不允許作為輸入等級。

目標等級不需要輸入百分比，固定視為：

```text
目標等級 0.000%
```

## 等級輸入規則

等級是否可輸入取決於目標設定的燃燒規則：

```text
無燃燒：260～300 每等都可輸入
燃燒 1+1（270）：260、262、264、266、268、270，270 後每等都可輸入
燃燒 1+1（280）：260、262、264、266、268、270、272、274、276、278、280，280 後每等都可輸入
```

如果輸入了目前燃燒規則下會被跳過的等級，系統不會儲存，避免把官方沒有計入的跳過等級也算進 EXP 需求。

## 經驗輸入規則

- 經驗輸入框使用 `type="text"` 搭配 `inputmode="decimal"`，避免部分手機瀏覽器 `type="number"` 在輸入小數點時產生小數點跑到前面或游標跳動的問題。
- 經驗輸入框只接受 0～100，最多小數點第三位。
- 玩家不用輸入 `%`，畫面右側會自動顯示百分比符號。
- 若輸入 `100`，系統在計算上會依目前燃燒規則視為「下一個有效等級 0.000%」。例如燃燒 1+1（270）時，`260 等 100` 等同 `262 等 0.000%`；無燃燒時，`260 等 100` 則等同 `261 等 0.000%`。
- 畫面會保留玩家原本輸入的等級與百分比，不會自動把紀錄文字改寫成下一等，避免造成誤會。

## 紀錄區顯示

每日紀錄表格會以「日期」彙整顯示，所以同一天可以輸入多筆紀錄，但下方列表預設只會看到每日一筆。

例如同一天新增：

```text
2026-06-28  262 等 0.000%  60 分
2026-06-28  264 等 10.000% 60 分
2026-06-28  264 等 25.000% 0 分，備註：活動獎勵
```

列表會彙整成：

```text
2026-06-28  結束 264 等 25.000%  本日總獲得合計  練等分鐘合計  練等效率
```

點每日列右側的「詳細」後，才會展開：

- 當日開始累積 EXP
- 當日結束累積 EXP
- 本日總獲得完整 EXP
- 納入效率 EXP
- 其他渠道 EXP
- 本日練等每小時完整 EXP
- 結束等級升級需求
- 備註彙整
- 當日每一筆原始輸入明細

每日列上的「刪除當日」會刪除該日期全部紀錄；詳細資訊裡的明細「刪除」只會刪除單筆原始紀錄。

每日紀錄表格預設只顯示短版 EXP，例如 `1.74 兆 EXP`，避免手機與 PC 版面被長數字撐開。上方統計卡片也會以短版 EXP 顯示，完整數字會放在滑鼠提示與詳細資訊中。


## 練等分鐘 0 的規則

如果某筆紀錄的「練等分鐘」輸入 `0`，系統會把它視為非練等來源，例如活動、任務、道具、秘藥或其他獎勵。

這種紀錄會：

```text
計入目前進度
計入距離目標與完成比例
計入每日總獲得 EXP
保留在當日明細與備註
```

但不會：

```text
增加累積練等時間
納入平均 EXP / 小時
納入剩餘需練時數的效率推算
納入每日最低時數的效率推算
```

因此玩家可以維持原本簡單紀錄方式：有練等就填實際分鐘，活動或其他渠道就填 `0` 並在備註標記來源。

## 目標日前統計邏輯

統計區不再單獨放一張「目標結束時間」卡片；目標時間仍保留在目標設定中，用來判斷推算完成日是否超過期限。

統計區不再把玩家視為每天 24 小時都在練等。

- 「剩餘練等時數」代表還需要實際練等多久，不是現實時間直接加上去。
- 「每日平均練等」會用有填練等分鐘的日期計算：累積練等分鐘 ÷ 有練等紀錄的天數 ÷ 60。0 分鐘的其他渠道紀錄不納入。
- 「目標每日 EXP」代表從現在到目標結束時間，平均每天需要獲得多少 EXP。
- 「每日最低時數」會用玩家目前的平均 EXP / 小時換算：

```text
每日最低時數 = 剩餘純練等時數 / 距離目標剩餘天數
```

例如還需要 40 小時純練等，距離目標剩 20 天，則每天至少需要練 2 小時。


## 同一天多筆紀錄排序

每日紀錄表單只有「日期」沒有精確時間，因此同一天新增多筆紀錄時，系統會用「結束進度」由低到高排序。

這樣玩家在同一天升等後再新增紀錄，例如先記 `262 等 0.000%`，再記 `264 等 10.000%`，最後補 `266 等 50.000%`，不會因為 randomUUID 或新增順序導致誤判成進度倒退。

不同日期仍然以日期先後為主，若後一天進度低於前一天，系統才會提示進度倒退。

## 自動儲存規則

只有「目標設定」會自動儲存，包含：

```text
起始等級
起始經驗
目標等級
目標結束時間
預設每日練等小時
```

玩家輸入後會自動驗證，合法才依目前儲存方式寫入資料並重新計算統計。若設定會造成既有紀錄倒退，系統不會覆蓋原本設定。

每日紀錄區仍然必須按「新增紀錄」才會新增資料；清除紀錄、重置全部、刪除單筆也都必須手動確認。

## 剩餘需練與推算完成日

「剩餘需練」代表純練等時間：

```text
累積平均 EXP 每小時 = 納入效率的 EXP / 實際練等分鐘 × 60
剩餘需練時數 = 剩餘 EXP / 累積平均 EXP 每小時
```

「預估完成日」會使用每日平均練等小時換算，並獨立顯示在統計卡片中：

```text
每日平均練等小時 = 累積練等分鐘 / 有練等紀錄的天數 / 60
推算完成天數 = 剩餘需練時數 / 每日平均練等小時
```

尚無練等紀錄時，系統會暫時使用目標設定裡的「預設每日練等小時」。開始輸入練等紀錄後，推算完成日會自動改用實際平均，不會永遠套用固定值，也不會把玩家當成每天 24 小時都在練等。

顯示格式會拆成兩張卡片，避免把長文字塞在小字裡：

```text
剩餘需練時數：654.8 小時
預估完成日：2027/05/21 下午08:59
說明：依實際每日平均 2.00 小時，約 327.4 天後完成
```

如果推算完成日期超過目標設定裡的「目標結束時間」，「預估完成日」日期與說明會以警告色顯示，並提示「預估超過目標時間」。


## 總 EXP 預估完成

「預估完成日」是用練等效率與每日平均練等小時推算，比較適合評估純練等進度。

「總 EXP 預估完成」則會使用每日總獲得 EXP 推算，包含練等與其他渠道成長：

```text
每日總獲得平均 = 已完成總 EXP / 有紀錄天數
總 EXP 推算天數 = 剩餘 EXP / 每日總獲得平均
```

這張卡片適合觀察「照目前每天整體成長速度，大概哪一天能達標」。如果活動或其他渠道很多，這個日期可能會比純練等推算更早；若推算結果超過目標結束時間，也會以警告色顯示。

## 100% 顯示規則

紀錄會保留玩家輸入的等級與百分比，但 EXP 計算會依照有效等級路線換算。

例如輸入 `260 等 100.000%`，畫面仍顯示 `260 等 100.000%`；但 EXP 計算會依目前燃燒規則判定下一個有效等級：

```text
無燃燒：260 等 100.000% = 261 等 0.000%
燃燒 1+1（270）：260 等 100.000% = 262 等 0.000%
燃燒 1+1（280）：278 等 100.000% = 280 等 0.000%
```


## 每日平均練等與預設值

目標設定裡的「預設每日練等小時」只在尚無紀錄時使用，讓工具一開始就能先估算完成日期。

只要開始輸入有練等分鐘的紀錄，系統就會改用玩家實際資料推估每日平均：

```text
每日平均練等小時 = 累積練等分鐘 / 有練等紀錄的天數 / 60
```

例如兩天分別練 `90` 分與 `150` 分：

```text
(90 + 150) / 2 / 60 = 2.00 小時 / 日
```

如果某天只有 `0` 分鐘的其他渠道紀錄，該天不會被算進每日平均練等天數，避免活動、任務、道具獎勵影響練等時間估算。

「每日最低時數」與「每日平均練等」是兩個不同概念：

```text
每日平均練等 = 玩家目前平均一天實際練多久
每日最低時數 = 依目前平均 EXP / 小時，要在目標日前達成，未來每天至少要練多久
```

## 介面修正

- 燃燒規則三選一卡片已固定尺寸與邊框，移除 hover 位移效果，避免滑鼠靠近選項時卡片抖動。

## 每日紀錄顯示順序

- 每日紀錄列表改為最新日期在最上方。
- 同一天仍可輸入多筆紀錄，列表會彙整成每日一筆。
- 紀錄區最多維持約 20 天列高的可視範圍，超過後會在紀錄區內垂直滑動。
- 詳細資訊仍可展開查看當日所有明細。

## 每週累積與每週平均推估

- 每週累積區塊固定以「週四到下週三」為一週。
- 同一週內的每日紀錄會自動彙整成一筆週資料。
- 每週累積會顯示：週期、週末進度、週總獲得 EXP、練等分鐘、週練等效率、其他渠道 EXP、紀錄數與備註。
- 統計區新增「每週平均預估完成」：以週四～週三的每週總獲得 EXP 平均後，再除以 7 換算每日平均，推算剩餘完成日期。
- 如果每週平均預估完成日超過目標結束時間，會以警告色顯示。

## 統計區排版

這版把原本全部混在一起的統計卡片重新拆成三個區塊：

```text
核心進度
- 目前進度
- 距離目標
- 完成比例
- 累積時間

練等效率推估
- 平均效率
- 目標每日 EXP
- 每日平均練等
- 依效率每日需練
- 剩餘需練時數
- 預估完成日

總成長推估
- 總 EXP 預估完成
- 每週平均預估完成
- 期限可達成比例：到目標日前，依每日平均與每週平均可補足剩餘 EXP 的比例；100% 代表可準時完成
```

每週累積區塊放在推估區下方、每日紀錄上方。這樣可以把「目前進度」、「練等效率推估」、「總 EXP / 每週平均推估」與「每週彙整」分開閱讀，避免統計卡片過多時混在一起。
